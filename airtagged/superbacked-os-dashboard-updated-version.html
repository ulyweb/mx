Flash ISO to USB

Write the snapshot ISO to your USB thumb drive with verification

🚨  This permanently overwrites the USB. Double-check your device. Your internal NVMe (nvme0n1) will NOT be touched.

How to Find Your USB Device Name

  1. Open terminal and run lsblk
  2. Find the drive that matches your USB size and has RM=1 (removable)
  3. Your internal drive is nvme0n1 — never use that
  4. USB is usually /dev/sda or /dev/sdb
  5. Enter just the device — /dev/sda not /dev/sda1

Step-by-Step Setup

Follow these in order — VM must be powered OFF before steps 2–4

Step 1 — Create the Shared Folder on Your MX Linux Host

  1. This creates the folder that will be shared with the VM.
  2. Generate the script below and run it as root — it creates the folder with correct permissions for libvirt-qemu to access it.
  3. The 777 permission is for simplicity. You can tighten it with ACLs later if needed.

Step 2 — Enable Shared Memory in virt-manager (VM must be OFF)

  1. Open Virtual Machine Manager
  2. Click on your VM — do not start it yet
  3. Click the Lightbulb icon (Show virtual hardware details)
  4. Click Memory in the left panel
  5. Check the box: Enable shared memory
  6. Click Apply

Step 3 — Add the virtiofs Filesystem Hardware (VM must be OFF)

  1. Still in the VM hardware details screen
  2. Click Add Hardware at the bottom left
  3. Select Filesystem from the list
  4. Set Driver to virtiofs
  5. Set Source path to your host folder — e.g. /home/uly/vm_share
  6. Set Target path to the tag name — e.g. share_tag (this is a label, not a real path)
  7. Click Finish

Step 4 — Start the VM and Mount the Share Inside It

  1. Power on the VM
  2. Open a terminal inside the VM
  3. The mount command uses the tag name you set — the script below generates the exact command.
  4. For auto-mount on every boot, add it to /etc/fstab inside the VM — the script handles this too.
setup-virtiofs.sh

Complete How-To Guide

Detailed instructions for every scenario — nothing skipped

🆕 Scenario A — Brand New Build (First Time)

  1. Log into your MX Linux machine as uly
  2. Open terminal → type su - → enter root password
  3. Go to Tab 02 — Build OS
  4. Fill in Root Password, VeraCrypt SHA256, Superbacked AppImage path
  5. Click Generate Script → Download Script
  6. In terminal: cd ~/Downloads && chmod +x build-custom-superbacked-os.sh
  7. Run: bash build-custom-superbacked-os.sh
  8. Wait 15–30 minutes for all 19 phases
  9. When done, customize the desktop then follow Scenario B

🎨 Scenario B — Customize Desktop Then Snapshot

  1. Log in as superbacked (auto-login)
  2. Set wallpaper, arrange icons, configure theme and taskbar
  3. Right-click each icon → Allow Launching to remove question mark
  4. When everything looks correct — close ALL open windows
  5. Open terminal → su - → enter root password
  6. Go to Tab 04 — Nuke MX Icons → Generate → Download → run it first
  7. Then go to Tab 03 — Pre-Snapshot → Generate → Download → run it
  8. At the end it asks: Launch mx-snapshot? → type y
  9. Set output folder: /home/uly/CustomOS-ISO → click Create Snapshot
  10. Wait 10–20 minutes for ISO to build
  11. Follow Scenario C to flash it

💣 Scenario C — Default MX Icons Keep Coming Back

  1. Go to Tab 04 — Nuke MX Icons
  2. Click Generate Nuke Script → Download
  3. Open terminal → su - → run: bash nuke-mx-icons.sh
  4. The script runs 5 passes — Pass 3 searches file content (catches renamed files)
  5. After nuke completes, immediately run the Pre-Snapshot script (Tab 03)
  6. Then take a new mx-snapshot and flash to USB

💾 Scenario D — Flash ISO to USB

  1. Plug in your 128GB USB drive
  2. Open terminal → run lsblk to find the USB device name
  3. Look for: size matches USB, RM=1 (removable). Usually /dev/sda
  4. Go to Tab 05 — Flash USB
  5. Enter ISO path and USB device → Generate → Download → run as root
  6. Script wipes, flashes, syncs, verifies sha256, then ejects
  7. Reboot → enter boot menu (F12/F2/ESC) → select USB
  8. Login: superbacked / superbacked

📂 Scenario E — Setting Up virtiofs Folder Sharing

  1. Go to Tab 06 — virtiofs Share
  2. Fill in host folder path, share tag name, guest mount point, and VM name
  3. Click Generate virtiofs Setup Script → Download → run as root on host
  4. The script creates the shared folder with correct permissions
  5. Then follow the manual virt-manager steps (shown on that tab) to enable shared memory and add the filesystem device
  6. Inside the VM, run the mount command generated by the script
  7. Files in the host folder instantly appear inside the VM

🛠 Troubleshooting

  • Default MX icons still showing → Tab 04 — Nuke MX Icons. Pass 3 reads file content to find renamed copies.
  • Terminal opens on every boot → Pre-snapshot script sets KDE loginMode=emptySession. Re-run it.
  • Icons have question mark → Right-click → Allow Launching. Pre-snapshot also auto-fixes this.
  • dd writes at 2+ GB/s → Wrong device. Run lsblk to confirm USB device name.
  • VeraCrypt needs administrator password → Enter root password, not user password.
  • superbacked user has sudo → Pre-snapshot script auto-removes from sudo group.
  • Network active on boot → Pre-snapshot verifies and re-enables airgap-lockdown service.
  • virtiofs not working → Confirm shared memory is enabled in VM Memory settings and VM was fully powered off before adding filesystem hardware.
